Что такое git diff ?
Расширенная команда git status 
Если git status отвечает на эти вопросы в самом общем виде,
перечисляя имена файлов, git diff показывает вам непосредственно добавленные и
удалённые строки — патч как он есть.

Важно отметить, что git diff сама по себе не показывает все изменения сделанные с
последнего коммита - только те, что ещё не проиндексированы. Такое поведение может
сбивать с толку, так как если вы проиндексируете все свои изменения, то git diff ничего не
вернёт

Другой пример: вы проиндексировали файл (название файла) и затем изменили его, вы
можете использовать git diff для просмотра как проиндексированных изменений в этом
файле, так и тех, что пока не проиндексированы

Сразу покажет не проиндексированный красным цветом и индексированный зеленым цветом

Как удалить файл ?
Команда git rm позволяет удалять отдельные файлы или группы файлов. Основное назначение git rm — удаление отслеживаемых файлов из раздела проиндексированных файлов Git. Кроме того, с помощью git rm можно удалить файлы одновременно из раздела проиндексированных файлов и рабочего каталога.

rm (Название файла) просто удалит файл 
К примеру у меня есть файл style.css и его нужно удалить, он не проиндексирован что нужно вводить 

rm style.css 
и файл удалиться 

Если вы изменили файл и уже проиндексировали его, вы должны использовать принудительное
удаление с помощью параметра -f

если у меня есть файл style.css и он проиндексирован то такая запись git rm -f style.css  удалит проиндексированный файл, style полностью удалится 

удалить файл из индекса, оставив его при этом в рабочем каталоге, как буд то его небыло 
Другими словами, вы можете захотеть оставить файл на жёстком диске, но перестать отслеживать изменения в нём как ?

git rm --cached (Название файла)
Убрать файл из последнего коммита, но не удалять этот файл.
Это особенно полезно, если вы забыли добавить что-то в файл .gitignore и по ошибке проиндексировали,
например, большой файл с логами, или кучу промежуточных файлов компиляции. Чтобы
сделать это, используйте опцию --cached

если есть индексированный файл stule.css то такая записю git rm --cached style.css удалит индекс и он будет опять крассным 

Убрать файл из последнего коммита и удалить этот файл. 
git rm index.js

Если прописать git status, будет следующее сообщение:
deleted: index.js

Поэтому можно изменить предыдущий коммит, убрав из него изменения:
git commit --amend --no-edit




Как переименовать файл ?
git mv index.html index_main.html  
он переименует файлы и не будет ругаться что он не проиндексирован, просто переменует 

это эквивалентно выполнению следующих команд
Представь что у тебя есть файл README.md
$ mv README.md README
$ git rm README.md
$ git add README

Git неявно определяет, что произошло переименование, поэтому неважно, переименуете вы
файл так или используя команду mv. Единственное отличие состоит лишь в том, что
mv — одна команда вместо трёх - это функция для удобства.


Как изменить закомиченный коментарий ?
Если вы хотите переделать
коммит — внесите необходимые изменения, добавьте их в индекс и сделайте коммит ещё
раз, указав параметр --amend:
git commit --amend
выйдет в терминале название коммита и его можно изменить вручную 

Очень важно понимать, что когда вы вносите правки в последний коммит,
вы не столько исправляете его, сколько заменяете новым, который
полностью его перезаписывает. В результате всё выглядит так, будто
первоначальный коммит никогда не существовал, а так же он больше не
появится в истории вашего репозитория.

Что такое git checkout ?
отмена изменений, например ты написал код и он тебе очень не нравиться,что бы не удалять ручками, пишешь git checkout и он откатывает все изменения к последнему коммиту
Важно понимать, что git checkout -- <file> — опасная команда. 

Помните, все что попало в коммит почти всегда Git может восстановить. Можно
восстановить даже коммиты из веток, которые были удалены, или коммиты,
перезаписанные параметром --amend (см. Восстановление данных). Но всё, что не было
включено в коммит и потеряно - скорее всего, потеряно навсегда

Итог команда тупо и навсегда удаляет все файлы которые ты писал и восстановить его невозможно 

В чем разница git reset и git restore ?
По сути, это альтернатива git reset,
которую мы только что рассмотрели. Начиная с версии 2.23.0, Git будет использовать git
restore вместо git reset для многих операций отмены

Важно понимать, что git restore <file> — опасная команда. Любые
локальные изменения, внесенные в этот файл, исчезнут — Git просто
заменит файл последней зафиксированной версией. Никогда не
используйте эту команду, если точно не знаете, нужны ли вам эти
несохраненные локальные изменения.

git remote rename.
Например, если вы хотите переименовать pb в paul, вы можете это сделать при помощи git remote rename

git remote rename (старое имя) (новое имя)

Стоит упомянуть, что это также изменит имена удалённых веток в вашем репозитории. То,
к чему вы обращались как pb/master, теперь стало paul/master.

как удалить удаленный репозиторий ?

При удалении ссылки на удалённый репозиторий все отслеживаемые ветки и настройки,
связанные с этим репозиторием, так же будут удалены.
